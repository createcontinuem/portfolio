EXPERIMENTS

Original hand-drawn artwork animated into living organisms, then grown iteratively across generations — each generation's final frame becoming the seed of the next. The resulting video lineage is displayed as a Three.js "digital garden": masked cutout sprites floating in 3D space with bioluminescent glow and particle systems.

The work is entirely original IP: no stock footage, no copyrighted characters, no personal photographs. Every frame is either a continuem drawing or a Runway-generated descendant of one.

Animate Fuse Grow Extract Render Mutate

Each generation drifts further from the source artwork. By gen03, the image is purely organism — no artwork origin visible. Generations stabilize after 4 iterations; subject stays organism-like but visual variety plateaus.

FULL LINEAGE · continuem × Runway Gen-4

Single mode

Reference image → resize to max 1280×720 → base64 data URI → gen4.5 image_to_video (5s) → download .mp4. Prompt source priority: --prompt arg → prompts.json per-image key → prompts.json._default.

Multi mode

2–3 reference images each encoded as base64, tagged ref0 / ref1 / ref2 → gen4_image text_to_image with @ref tags in prompt (5 credits) → composite image → gen4.5 image_to_video on composite URL (5s, 60 credits). Combo config: references/images/photos/multi.json

Growth mode

Seed .mp4 → for each iteration: cv2 extracts last frame → without reference: gen4.5 image_to_video on last frame (60 credits) · with reference: gen4_image blends last frame + reference (5 credits) → composite URL → gen4.5 image_to_video (60 credits). Chain breaks on first failure. Each iteration: video → last frame → video.

for i in 1..N: frame = extract_last_frame(video) # OpenCV if reference: composite = text_to_image(frame, ref) # 5 cr seed = composite video = image_to_video(seed, prompt) # 60 cr
grow.py core loop. Each iteration 60–65 credits. CloudFront output URLs expire 24–48h — always download to local .mp4 immediately.
◈
Garden Pipeline
garden.py scans videos/ and growth/ → assigns 3D grid positions (spacing=3.5, ±jitter) → assigns float speed, amplitude, phase per organism → writes garden_state.json. extract_masks.py picks sharpest frame (Laplacian variance) across 25/50/75% candidates → rembg removes background → transparent PNG.
◎
Background Removal
Rectangular video planes looked like copy-pasted screens. rembg extracts subject per video → alpha mask → Three.js alphaMap. Result: organisms float as shaped cutouts with glow halo. Three.js: MeshBasicMaterial with map + alphaMap + transparent:true + depthWrite:false.
↺
Orbit Camera
OrbitControls with autoRotate at speed 0.4. Each sprite gets a glow Sprite child (glowMaterial, scale 3×3). Slow auto-rotate with free user orbit. Positions from spiral_layout(i) distribute organisms in 3D space.

Experiments run May 8–12, 2026. Prompt generalization: each artwork's color palette drives its own distinct organism character. Running total ~915 credits of 50,000 participant allocation.

✓  What worked
Multi-fuse as primary seed
Composite fuses visual DNA of both artworks. More interesting than single-image mode. soul_fusion_239_247.mp4 used as primary seed for growth chain — downstream generations inherit blended character from both source artworks.
Reference blending for visual variation
Same seed and reference (cambodia-ruins.jpg), different prompt intent produces clearly distinct visual direction: plant_garden achieves bioluminescent botanical aesthetic; ruins_mutation produces more aggressive mutation aesthetic. Reference photo introduces architectural stone texture into organic growth.
Experiment folder scoping
All scripts accept --experiment <name> to scope outputs under output/experiments/<name>/. Separates videos/, composites/, growth/, masks/, results.json, growth_log.json per experiment. --build-only flag separates state build from serving.
✗  What to push further
Microorganism prompt too abstract
Default experiment: visually interesting but not reading as actual microbiology. Prompt generalized well across all 7 artworks but produced abstract results rather than convincing microscopic organisms. Bacteria prompt in gen03_garden addresses this with dark field microscopy framing.
Generation variety plateaus
gen03_garden (4 iterations): chained growth pipeline working, generations look similar across iterations. gen03_gen01 identified as best branch point for reference-blended experiments rather than continuing the chain further. Visual variety requires reference injection to maintain drift.
Runway API interpolation is generative
interpolate.py using text_to_image composite → image_to_video: generative/hallucinated, not a real interpolation. Local alternatives tested: crossfade (clean dissolve, no motion), Farneback optical flow (also reads as dissolve for dissimilar images), homography + Delaunay mesh morph evaluated separately.
01 Animation API
Runway Gen-4 / Gen-4.5 — image_to_video and text_to_image. Python SDK: runwayml v4.14.0. API key in .env as RUNWAYML_API_SECRET (must start with key_).
02 Image & Video Processing
opencv-python for frame extraction (frame N-1 via cv2). Pillow for resize + JPEG encode before base64. rembg for background removal → transparent PNG alpha sprites. Python 3.12.
03 3D Garden Viewer
Three.js with OrbitControls, MeshBasicMaterial, Sprite. Served via Python http.server on localhost:8080 with CORS enabled. Organisms rendered as floating video planes with alphaMap cutouts and Sprite glow halos. Auto-rotate speed 0.4.

Technical Notes

Constraints encountered and resolved during the hackathon. Each required a design decision that shaped the final pipeline architecture.

Constraint
Value / Behavior
Resolution
image_to_video input
Single prompt_image only — no multi-image support
Multi-image: use text_to_image composite first, then feed composite URL to image_to_video
reference_images tag format
Non-obvious: object key = ref0, prompt text = @ref0
Discovered via trial. grow.py --reference flag constructs @ref0 @ref1 prompt prefix automatically
Large image → HTTP 413
Payload too large for base64-encoded high-res images
Auto-resize to 1280×720 max, JPEG 85% quality before encoding. Applied in generate_videos.py and grow.py
CloudFront output URLs
Output video URLs expire 24–48h after generation
Always download to local .mp4 immediately after poll_until_complete returns. Never use URL as next-gen seed
prompt_text max
1000 characters max
Truncate before API call: prompt_text=prompt[:1000]
Duration range
2–10 seconds, all experiments use 5s
5s balances credit cost (60 cr) and enough motion for growth chain. Longer durations give more frame candidates for extract_masks.py
Supported ratios
1280:720 · 720:1280 · 1104:832 · 832:1104 · 960:960 · 1584:672
All experiments use 1280:720. Auto-resize target matches this ratio to avoid cropping artifacts
IP & Identity Decisions
No copyrighted characters
No Pikachu, Hello Kitty, anime IP, or other licensed characters. Conflicts with judging criteria and creates moderation risk.
No personal photos or videos
Runway ToS does not exclude uploads from training. All source artwork is original continuem brand IP. Reference photos (cambodia-ruins.jpg) used as style influence, not subjects.
All output is original IP
Every frame is either a continuem drawing or a Runway-generated descendant of one. Own the full output chain outright.
